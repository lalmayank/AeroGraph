# AeroGraph OTLP Import Bridge

The OTLP Import Bridge provides a lossless round-trip mechanism and a robust heuristic fallback for translating OpenTelemetry (OTLP) Spans back into canonical AeroGraph `TraceEvent` objects. This allows traces generated by other GenAI observability systems or external OpenTelemetry SDKs to be imported into AeroGraph seamlessly.

## Architecture

The Import Bridge operates symmetrically to the Export Bridge, transforming an `OtlpSpan` to a `TraceEvent`. It is exposed in both TypeScript (`@aerograph/otel`) and Python (`aerograph-otel`).

The import flow is as follows:
1. `importOtlpToEvents` extracts individual `OtlpSpan` records from a standard `OtlpExportRequest`.
2. Each span goes through `resolveAeroGraphKindFromSpan` to determine the corresponding AeroGraph `TraceEventKind`.
3. An actor ID is resolved from attributes, defaulting to a fallback `defaultActorId` if not present.
4. Attributes are mapped to the canonical `TraceEvent` payload.
5. Missing or external spans fall back safely to a `note` event, wrapping all original data without dropping telemetry.

## Mapping Heuristics (OTel Span → AeroGraph Event)

When an imported span contains an `aerograph.kind` attribute, the mapping is lossless. For foreign spans, the following heuristic sequence determines the AeroGraph kind:

| OTel Span Name / Attribute | AeroGraph Kind |
|---|---|
| `aerograph.kind` attribute present | Use that kind directly |
| name contains `gen_ai.chat` OR `gen_ai.operation.name=chat` | `response` (with synthetic logic applied later) |
| `gen_ai.tool.name` + `gen_ai.tool.call.id` | `tool_call` |
| `aerograph.kind=tool_result` | `tool_result` |
| `error.type` or status code=error | `error` |
| `gen_ai.operation.name=retrieve` | `retriever` |
| All others | `note` (with foreign attributes mapped to payload) |

## Limitations

1. **Information Loss (Streaming)**: Some granular streaming metrics (like TTFT and tokens per second) may be discarded if not explicitly specified in the incoming OTLP span.
2. **One-Way ID Preservation**: AeroGraph's natively generated `traceId` and `spanId` map cleanly to OTel IDs without modification, but arbitrary external OTel `traceId`s are preserved "as is". The user is responsible for ensuring no accidental collisions occur when injecting external IDs directly.
3. **Point-in-Time Discrepancy**: AeroGraph traces are instantaneous points (`occurredAt`). OTel spans have `startTime` and `endTime`. The import bridge exclusively maps `startTimeUnixNano` back to `occurredAt` — discarding the synthetic duration generated on export.

## Examples

### TypeScript
```typescript
import { importOtlpToEvents } from '@aerograph/otel';
import type { OtlpExportRequest } from '@aerograph/otel';

const otlpPayload: OtlpExportRequest = {
  resourceSpans: [
    /* ... OTLP Payload ... */
  ]
};

const events = importOtlpToEvents(otlpPayload, {
  defaultActorId: "external-system",
  preserveOriginalIds: true
});

// `events` is an array of sorted canonical TraceEvents.
```

### Python
```python
from aerograph_otel.import_ import import_otlp_to_events

otlp_payload = {
    "resourceSpans": [
        # ... OTLP Payload ...
    ]
}

events = import_otlp_to_events(
    otlp_payload,
    default_actor_id="external-system",
    preserve_original_ids=True
)

# `events` contains a list of canonical TraceEvent dicts sorted by timestamp.
```
